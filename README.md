# 두벌식 자판 초중성 모아치기 개발 문서

## 프로젝트 개요

두벌식 자판 초중성 모아치기는 표준 두벌식 개선 자판인 [두겹이(두벌식 겹받침 e) 자판](https://blog.naver.com/eekdland/224142632310)과 표준 두벌식 응용 자판인 [두줄이(두벌식 줄맞춤 e) 자판](https://blog.naver.com/eekdland/224252538464)의 **두벌식 자판 위에서 부분적인 동시 입력(chord)을 지원하는 실험적 입력기**입니다.

목표는 다음과 같습니다.

- 기존 두벌식 사용자가 **추가 학습이 거의 없이 사용할 수 있는 입력기**
- 동시 입력에서 **초성 + 중성의 부분 모아치기 지원**
- 일반 타이핑과 충돌하지 않는 **속도 기반 chord 인식**

구현은 **AutoHotkey 기반 입력 인터셉트 방식**으로 이루어집니다.

(AutoHotkey 코드 구현은 AI 도움을 받아 작성되었습니다.)

---

## 설계 목표

두벌식 자판 모아치기는 다음 원칙을 기반으로 설계되었습니다.

### 1. 두벌식 체계를 유지한 두겹이 자판과 두줄이 자판을 사용

키 배열은 기존 두벌식 개선 자판인 두겹이 자판과 응용 자판인 두줄이 자판을 사용합니다.

```
ㄱ ㄴ ㄷ ...
ㅏ ㅓ ㅗ ...
```

따라서 기존 두벌식 사용자에게 **학습 비용이 적습니다.**

---

### 2. 속도 기반 chord 입력

동시 입력은 **시간창(time window)** 으로 판단합니다.

```
입력 간격 ≤ chord window → chord
입력 간격 > chord window → 일반 이어치기
```

기본값

```
chord window = 25 ms
```

이 값은 일반 타이핑과 충돌하지 않으면서도

빠른 동시 입력을 chord로 인식하기 위한 값입니다.

---

### 3. V-start 기반 재정렬

(이 문서에서는 모음을 V로, 자음을 C로 표시합니다.)

두벌식에서는 자음이 **초성/종성 역할을 동시에 가지기 때문에**

세벌식처럼 단순한 chord 처리로는 음절을 구성할 수 없습니다.

그래서 두벌식 자판 초중성 모아치기는 **모음 중심 입력 구조**를 사용합니다.

입력 흐름:

```
V 입력
→ chord window 동안 입력 수집
→ 자모 재정렬
→ IME 전달
```

이 방식은 **음절 핵(nucleus) 중심 입력 모델**입니다.

---

## 입력 처리 구조

전체 입력 흐름은 다음과 같습니다.

```
키 입력
↓
AHK 인터셉트
↓
chord window 버퍼
↓
패턴 분석
↓
자모 재정렬
↓
IME로 전송
```

---

### chord 버퍼

버퍼에는 chord window 동안 입력된 키가 저장됩니다.

예

```
[k][w]
```

이 경우 입력 간격이 window 이내라면

```
VC
```

패턴으로 처리됩니다.

---

### 패턴 분석

버퍼의 입력 패턴을 다음 기준으로 분류합니다.

```
V  = 모음
C  = 자음
```

대표 패턴:

| 패턴 | 의미 |
| --- | --- |
| VC | 모음 → 자음 |
| VCV | 모음 → 자음 → 모음 |
| VVC | 모음 → 모음 → 자음 |

---

### 패턴 정규화

두벌식 자판 모아치기는 다음 패턴을 처리합니다.

#### VC → CV

```
ㅏ + ㄴ
→ ㄴ + ㅏ
```

---

#### VCV → CVV

예

```
ㅏ ㄴ ㅗ
→ ㄴ ㅗ ㅏ
```

---

#### VVC → CVV

예

```
ㅏ ㅗ ㄴ
→ ㄴ ㅗ ㅏ
```

---

이 정규화는 IME에 **정상적인 음절 순서**를 전달하기 위한 것입니다.

---

## 시간창 설계

현재 기본 설정

```
chord window = 25 ms
```

이 값은 다음을 고려하여 선택되었습니다.

- 일반 이어치기 오인식 최소화
- 동시 입력 인식 안정성
- 평균 타속 대응

대략적인 기준:

| 값 | 의미 |
| --- | --- |
| 20 ms | 매우 보수적 |
| 25 ms | 기본값 |
| 30 ms | 관대한 설정 |

---

## 왜 V-start 구조인가

두벌식 자판에서는 자음이 다음 역할을 모두 수행합니다.

```
초성
종성
다음 음절 초성
```

따라서 자음 시작 chord는 음절 경계를 결정하기 어렵습니다.

반면 모음은 항상 **음절 핵**이므로 다음 구조를 사용합니다.

```
V 입력
→ chord window 동안 입력 수집
→ 자모 재정렬
→ IME 전달
```

이 방식은 음절 경계 문제를 단순화합니다.

---

## 세벌식과의 관계

세벌식 자판은 **자모 분리 구조**를 기반으로 설계되어

모아치기가 자연스럽게 가능합니다.

두벌식 자판 초중성 모아치기는 세벌식과 같은 구조를 사용하지 않으며

두벌식 환경에서 **부분적인 chord 입력을 실험적으로 구현한 입력기**입니다.

즉

```
세벌식 → 구조 기반 모아치기
두벌식 → 시간창 기반 부분 모아치기
```

입니다.

---

## 세모이 자판 초기 구조와의 관계

두벌식 자판 초중성 모아치기의 입력 모델은 **[세모이 자판](https://blog.naver.com/eekdland/220526834927) 초기 설계와 유사한 부분**이 있습니다.

세모이 자판 초기 버전은 다음 구조를 사용했습니다.

```
시간창 동안 자모 수집
→ 자모 재정렬
→ 음절 생성
```

두벌식 자판 초중성 모아치기는 이를 **두벌식 환경에 맞게 제한적으로 적용**한 구조입니다.

차이점:

| 입력기 | 구조 |
| --- | --- |
| 세모이 초기 | 초중종 전체 재정렬 |
| 두벌식 자판 모아치기 | V-start 기반 재정렬 |

즉 두벌식 자판 모아치기는

```
시간창 기반 음절 재구성
```

이라는 아이디어를 두벌식 환경에 적용한 입력기입니다.

## 안정성 고려

다음 요소들이 안정성 확보를 위해 사용됩니다.

### chord timeout

버퍼가 일정 시간 이상 유지되지 않도록 제한합니다.

```
ChordTimeout
```

---

### buffer length 제한

버퍼가 비정상적으로 길어지는 것을 방지합니다.

```
max buffer length
```

---

### 이중모음 일부 조합과 겹받침을 IME에서 처리

일부 이중모음 조합과 겹받침 조합은 IME 레벨에서 처리합니다.

예

```
ㅓ + ㅜ → ㅝ
ㅁ + ㄹ → ㄻ
```

---

## 제한 사항

- AutoHotkey 기반이므로 일부 프로그램에서 입력 지연이 발생할 수 있습니다.
- 날개셋 입력기 IME에 의존성을 갖고 있어서 독립적으로 사용하기에는 한계가 있습니다.
- 극단적으로 빠른 타이핑에서는 chord 오인식이 발생할 수 있습니다.
- 표준 두벌식 배열 자체의 인체공학적 한계는 해결하지 않습니다. 이 한계는 세벌식 자판이 해결하는 부분입니다.
